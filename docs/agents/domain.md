# Domain Docs

**Layout:** Single-context

## Where to look

| Resource | Path |
|---|---|
| Domain language & project context | `CONTEXT.md` (repo root) |
| Architectural decision records | `docs/adr/` |

## Consumer rules

- Read `CONTEXT.md` before diagnosing bugs, writing tests, or proposing architectural changes.
- Check `docs/adr/` for past decisions before proposing a design that may have been considered and rejected.
- If `CONTEXT.md` doesn't exist yet, proceed without it and suggest creating one.
- If `docs/adr/` doesn't exist yet, there are no recorded decisions yet — that's fine.
